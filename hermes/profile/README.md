# Hermes Product Teams Profile

This directory is the runnable Hermes Product Teams agent package. It is intended to be installed as a named Hermes profile called `product-teams`.

## What this profile does

Hermes Product Teams is a **Discovery + Living Docs Agent** for product teams. It turns messy product inputs into source-linked product-memory artifacts:

- discovery notes;
- customer insights;
- decision-log entries and proposals;
- open questions;
- PRD update proposals;
- weekly product briefs.

It does **not** silently edit source-of-truth docs, create tickets, send stakeholder messages, or change external systems. Important PRD/spec changes are proposed first and require human approval.

## Install locally

From the repository root:

```bash
python3 scripts/install_profile.py --workspace examples/workspace
```

For a new real workspace, add `--init-workspace` to create the standard artifact folders and starter Markdown files without overwriting existing artifacts:

```bash
python3 scripts/install_profile.py \
  --workspace /absolute/path/to/product-workspace \
  --init-workspace
```

To install under a different Hermes profile name, pass `--profile-name <name>` using a Hermes-compatible lowercase slug such as `acme-product-memory`; the installer renders that name into both `config.yaml` and the packaged runtime scripts so dry-runs and real captures invoke the matching profile.

This creates:

```text
~/.hermes/profiles/product-teams/
├── config.yaml
├── SOUL.md
├── distribution.yaml
├── workflows/
│   ├── capture-input.md
│   ├── weekly-brief.md
│   └── review-prd-proposals.md
├── skills/product-team-memory/SKILL.md
└── scripts/
    ├── run_agent_capture.py
    └── run_weekly_brief.py
```

No credentials are copied or generated by this installer. Configure model/provider credentials for the profile separately if your Hermes setup does not inherit them:

```bash
hermes --profile product-teams model
# or use your normal Hermes auth/provider setup, then run with --profile product-teams
```

## Run the actual agent

If you installed the profile into `~/.hermes/profiles/product-teams/`, dry-run the packaged capture command like this:

```bash
python3 ~/.hermes/profiles/product-teams/scripts/run_agent_capture.py \
  --input /absolute/path/to/customer-feedback.md \
  --dry-run
```

Run through Hermes:

```bash
python3 ~/.hermes/profiles/product-teams/scripts/run_agent_capture.py \
  --input /absolute/path/to/customer-feedback.md
```

You can also capture pasted text without saving a file first, either with `--text` or by piping content via stdin (the two are alternatives to `--input`; supplying both `--input` and `--text` is an error):

```bash
python3 ~/.hermes/profiles/product-teams/scripts/run_agent_capture.py \
  --text "Slack thread: two customers say weekly CSV exports keep timing out" \
  --dry-run

pbpaste | python3 ~/.hermes/profiles/product-teams/scripts/run_agent_capture.py --dry-run
```

Pasted/stdin content is embedded in the capture prompt between `BEGIN/END PASTED INPUT` markers, and the agent records the source as user-pasted text with today's date when the text doesn't name its own source.

The script invokes `hermes chat --profile product-teams --skills product-team-memory ...` with a self-contained capture prompt and defaults to the workspace configured in `~/.hermes/profiles/product-teams/config.yaml`.

If you are developing from the repo checkout instead of the installed profile, use:

```bash
python3 scripts/run_agent_capture.py \
  --input examples/inputs/001-customer-feedback-thread.md \
  --workspace examples/workspace \
  --dry-run
```

## Weekly brief

After capturing product inputs, generate the weekly brief with the packaged runner:

```bash
python3 ~/.hermes/profiles/product-teams/scripts/run_weekly_brief.py \
  --dry-run

python3 ~/.hermes/profiles/product-teams/scripts/run_weekly_brief.py
```

If you are working from the repo checkout instead of the installed profile, use:

```bash
python3 scripts/run_weekly_brief.py \
  --workspace examples/workspace \
  --dry-run

python3 scripts/run_weekly_brief.py \
  --workspace examples/workspace
```

## Profile files

- `SOUL.md` — the agent persona, scope, safety rules, and output contract.
- `distribution.yaml` — profile package metadata for Hermes-compatible distribution.
- `config.example.yaml` — profile-local defaults and workspace path placeholder.
- `workspace.example.yaml` — artifact paths the agent should maintain.
- `workflows/` — reusable prompts for capture, weekly briefing, and PRD proposal review.
