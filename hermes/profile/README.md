# Hermes Product Teams Profile

This directory is the runnable Hermes Product Teams agent package. It is intended to be installed as a named Hermes profile called `product-teams`.

## What this profile does

Hermes Product Teams is a **Discovery + Living Docs Agent** for product teams. It turns messy product inputs into source-linked product-memory artifacts:

- discovery notes;
- customer insights;
- decision-log entries and proposals;
- open questions;
- PRD update proposals;
- weekly product briefs.

It does **not** silently edit source-of-truth docs, create tickets, send stakeholder messages, or change external systems. Important PRD/spec changes are proposed first and require human approval.

## Install locally

From the repository root:

```bash
python3 scripts/install_profile.py --workspace examples/workspace
```

This creates:

```text
~/.hermes/profiles/product-teams/
├── config.yaml
├── SOUL.md
├── distribution.yaml
├── workflows/
│   ├── capture-input.md
│   ├── weekly-brief.md
│   └── review-prd-proposals.md
└── skills/product-team-memory/SKILL.md
```

No credentials are copied or generated by this installer. Configure model/provider credentials for the profile separately if your Hermes setup does not inherit them:

```bash
hermes --profile product-teams model
# or use your normal Hermes auth/provider setup, then run with --profile product-teams
```

## Run the actual agent

Dry-run the command first:

```bash
python3 scripts/run_agent_capture.py \
  --input examples/inputs/001-customer-feedback-thread.md \
  --workspace examples/workspace \
  --dry-run
```

Run through Hermes:

```bash
python3 scripts/run_agent_capture.py \
  --input examples/inputs/001-customer-feedback-thread.md \
  --workspace examples/workspace
```

The script invokes `hermes chat --profile product-teams --skills product-team-memory ...` with a self-contained capture prompt.

## Weekly brief

After capturing product inputs, generate the weekly brief with the packaged runner:

```bash
python3 scripts/run_weekly_brief.py \
  --workspace examples/workspace \
  --dry-run

python3 scripts/run_weekly_brief.py \
  --workspace examples/workspace
```

## Profile files

- `SOUL.md` — the agent persona, scope, safety rules, and output contract.
- `distribution.yaml` — profile package metadata for Hermes-compatible distribution.
- `config.example.yaml` — profile-local defaults and workspace path placeholder.
- `workspace.example.yaml` — artifact paths the agent should maintain.
- `workflows/` — reusable prompts for capture, weekly briefing, and PRD proposal review.
