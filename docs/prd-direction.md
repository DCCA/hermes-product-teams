# PRD Direction

## Direction decision

Hermes Product Teams should start as a **Discovery + Living Docs Agent**: a permission-aware product team memory layer that keeps discovery, decisions, and product docs current from messy inputs.

This is the direction to protect while improving the MVP.

## Product thesis

Product teams already have roadmap, issue, research, and docs tools. The gap is continuity: customer evidence, decisions, open questions, and PRD/spec changes drift apart across meetings, chats, docs, and tickets.

Hermes Product Teams should own the **memory/discovery/docs layer**, not the whole product-management stack.

## MVP user

A PM, founder, or product trio that already has messy product inputs and wants faster synthesis without losing source evidence or decision history.

## What the MVP should do

- Capture messy product inputs such as customer feedback, meeting notes, support snippets, and PM thoughts.
- Convert those inputs into structured discovery notes.
- Maintain source-linked customer insights.
- Preserve decision proposals and rationale.
- Propose PRD/spec changes with explicit source evidence.
- Produce weekly product briefs that summarize discovery signals, decisions, open questions, and proposed next actions.

## What the MVP should not do

- It is **not a roadmap management tool**.
- It is **not a user research platform**.
- It should not replace Productboard, Jira Product Discovery, Linear, Jira, Dovetail, Maze, Sprig, Notion, or Confluence.
- It should not silently edit important source-of-truth docs.
- It should not create issues, send stakeholder messages, or change external tools without explicit approval.
- It should not claim revenue-weighted prioritization or product analytics without integrations.

## Trust model

The core trust promise is:

> Source-linked evidence, proposed changes, and human approval before important docs change.

Generated outputs should separate:

- facts;
- assumptions;
- evidence;
- decisions;
- open questions;
- recommendations;
- next actions.

## PRD alignment checks

Every new workflow should answer:

1. Does this improve discovery, decision memory, living docs, or weekly product communication?
2. Does it preserve source-linked evidence?
3. Does it propose important PRD/spec changes instead of silently applying them?
4. Does it avoid expanding into roadmap management, user research operations, or project management too early?
5. Does it remain useful with a local/private Markdown workspace before requiring SaaS integrations?

If the answer to any question is no, the feature is probably outside the MVP direction.
